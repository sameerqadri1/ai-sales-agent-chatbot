## 📜 Code of Conduct

This project adheres to an open and welcoming community environment. Please treat all contributors with respect, courtesy, and constructive communication.

---

## 💡 How Can I Contribute?

### Reporting Bugs

Before creating a bug report, please check existing GitHub issues. When opening an issue, please include:
- A clear, descriptive title.
- Steps to reproduce the issue.
- Expected vs. actual behavior.
- Relevant system specifications (Python version, OS, WooCommerce plugin versions).

---

### Suggesting Enhancements

Feature requests are always welcome! Please explain:
- The problem your enhancement solves.
- Proposed implementation or user workflow.
- Any alternative solutions considered.

---

### Submitting Pull Requests

1. Fork the repository on GitHub.
2. Create a topic branch from `main`:
   ```bash
   git checkout -b feature/my-new-feature
   ```
3. Make your changes and test locally.
4. Commit your changes with clear, descriptive commit messages:
   ```bash
   git commit -m "feat: Add support for custom coupon prefix configuration"
   ```
5. Push your branch to your fork and submit a Pull Request to `main`.

---

## 🧹 Development & Coding Standards

- **Python**: Follow PEP 8 style guidelines. Use type hints (`typing`) where appropriate.
- **JavaScript**: Use clean Vanilla ES6+ without unnecessary external framework dependencies for the widget.
- **Documentation**: Update relevant `docs/` files if your changes alter API schemas or environment configurations.
